Install

Installing the plugin

Creative Studio installs as a Claude Code plugin marketplace — one marketplace, four plugins (the studio plus its Airtable, Figma, and Frame.io connections). Add it once and everything is available in any session.

Installing the plugin Video coming soon
1

Add the marketplace

Point Claude Code at the Creative Studio marketplace — a shared path while we pilot, a git URL once it ships.

Claude Code
/plugin marketplace add <path-or-git-url>
2

Install the four plugins

The studio plugin plus the three connections it composes — Airtable, Figma, and Frame.io.

Claude Code
/plugin install creative-studio@creative-studio
/plugin install airtable@creative-studio
/plugin install figma@creative-studio
/plugin install frameio@creative-studio
Also needed: the Anthropic document skills — docx, xlsx, pptx, pdf, deloitte-brand, deloitte-pptx — from the standard Anthropic skills marketplace. The studio composes them for every document and deck it renders.
3

Open the control room

Skills load on their own when a task calls for them. /cs shows where every project sits and what runs next; the phase commands are typed shortcuts.

The commands
/cs   /cs-intake   /cs-pre-production   /cs-production   /cs-post-production   /cs-wrap
Connections

Signing in to Airtable, Figma & Frame.io

The core works with no setup at all — it runs against your local OneDrive-synced folder. Three connections light up the pipeline, design, and the review loop.

No setup

OneDrive (local)

Works out of the box against your local synced folder. No connector, no cloud auth. This is the working surface for every document.

Sign in once

Airtable

The pipeline and system of record. The first skill that touches it opens the sign-in flow; approve it once.

Sign in once

Figma

Design generation and design-to-code, driven from the brief. Same one-time sign-in.

Adobe token

Frame.io

The cut-review loop, media, and the archive. Add your Adobe token once and the review skills go live.
